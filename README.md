1. 定期自動執行（不用管它）
因為我們在腳本裡設定了 cron: '0 2 * * 1'，GitHub 系統會在 每週一的凌晨 2 點 (UTC 時間) （相當於台灣時間每週一早上 10 點）自動啟動機器幫您爬取資料。您什麼都不用按，時間到它就會自己跑。

2. 手動立即執行（現在馬上測試！）
因為腳本裡有加上 workflow_dispatch: 這個神奇的咒語，它允許您隨時手動點擊按鈕叫它立刻開工。如果您現在就想測試有沒有設定成功，請依照以下步驟操作：

在您的 GitHub 專案首頁，點擊畫面上方選單的 「Actions」 標籤（在 Code 和 Pull requests 旁邊）。

看向畫面左側的選單，您會看到剛剛命名的 「自動化 RDKS 爬蟲」，點擊它。

畫面的右邊會出現一個帶有藍色橫幅的區塊，上面有一個寫著 「Run workflow」 的按鈕。

點擊 「Run workflow」，然後在彈出的選單中再點擊一次綠色的 「Run workflow」。

稍微等個十幾秒，畫面下方就會出現一個正在轉圈圈的任務，這代表您的雲端機器人已經開始跑爬蟲程式了！

等任務旁邊出現綠色的打勾（✅）後，點進去任務，滑到最下面找到 「Artifacts」，就能下載熱騰騰的爬蟲結果檔案囉！快去試試看！
